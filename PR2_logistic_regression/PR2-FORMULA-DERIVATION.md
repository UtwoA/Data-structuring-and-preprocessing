# Вывод формул "как у доски" для защиты лабораторной №2

Этот файл нужен для ситуации, когда преподаватель просит не просто рассказать идею,
а именно последовательно вывести формулы.

Здесь разложено:

1. как из логистической модели получается вероятность;
2. как записывается правдоподобие;
3. как из него получается `log-loss`;
4. как из `log-loss` получается градиент;
5. как из этого следует шаг градиентного спуска;
6. как записывается критерий CFS.

---

## 1. Начинаем с модели логистической регрессии

Пусть:

- `x = (x_1, x_2, ..., x_n)` — вектор признаков;
- `w = (w_1, w_2, ..., w_n)` — веса;
- `b` — свободный коэффициент.

Сначала строим линейную комбинацию:

`z = b + w_1 x_1 + w_2 x_2 + ... + w_n x_n`

Короткая запись:

`z = b + w^T x`

Теперь применяем сигмоиду:

`sigma(z) = 1 / (1 + e^{-z})`

Тогда вероятность класса `1`:

`p = P(y = 1 | x) = sigma(z)`

И вероятность класса `0`:

`P(y = 0 | x) = 1 - p`

Это исходная вероятностная модель.

---

## 2. Вероятность одного наблюдения

Пусть для одного объекта истинный класс `y` может быть только `0` или `1`.

Тогда:

- если `y = 1`, вероятность наблюдения равна `p`;
- если `y = 0`, вероятность наблюдения равна `1 - p`.

Оба случая можно записать одной формулой:

`P(y | x) = p^y (1 - p)^{1 - y}`

Проверка:

### Случай `y = 1`

`P(y | x) = p^1 (1 - p)^0 = p`

### Случай `y = 0`

`P(y | x) = p^0 (1 - p)^1 = 1 - p`

Это очень важная формула: она связывает бинарную метку и вероятность модели.

---

## 3. Правдоподобие всей обучающей выборки

Пусть есть выборка:

`(x_1, y_1), (x_2, y_2), ..., (x_m, y_m)`

Предполагаем, что наблюдения независимы.

Тогда общее правдоподобие равно произведению вероятностей всех наблюдений:

`L(w, b) = Product(i = 1..m) P(y_i | x_i)`

Подставляем формулу для одного объекта:

`L(w, b) = Product(i = 1..m) [ p_i^{y_i} (1 - p_i)^{1 - y_i} ]`

Где:

- `p_i = P(y_i = 1 | x_i) = sigma(b + w^T x_i)`

Идея метода максимального правдоподобия:

- нужно выбрать `w` и `b`, при которых `L(w, b)` максимально.

---

## 4. Переход к логарифму правдоподобия

С произведением работать неудобно, поэтому берём логарифм:

`log L(w, b) = log Product(i = 1..m) [ p_i^{y_i} (1 - p_i)^{1 - y_i} ]`

Используем свойство логарифма:

`log(ab) = log a + log b`

и

`log(a^k) = k log a`

Тогда:

`log L(w, b) = Sum(i = 1..m) log [ p_i^{y_i} (1 - p_i)^{1 - y_i} ]`

`log L(w, b) = Sum(i = 1..m) [ y_i log p_i + (1 - y_i) log(1 - p_i) ]`

Это логарифм правдоподобия.

Почему это удобно:

- вместо произведения получаем сумму;
- точка максимума не меняется;
- легче брать производные.

---

## 5. Переход к функции потерь

В оптимизации обычно удобнее минимизировать, а не максимизировать.

Поэтому берут отрицательный логарифм правдоподобия:

`- log L(w, b) = - Sum(i = 1..m) [ y_i log p_i + (1 - y_i) log(1 - p_i) ]`

Чтобы получить среднюю ошибку на объект, делят на `m`:

`J(w, b) = - (1 / m) * Sum(i = 1..m) [ y_i log p_i + (1 - y_i) log(1 - p_i) ]`

Это и есть функция потерь:

- `log-loss`
- `binary cross-entropy`

Ключевой вывод:

максимизация правдоподобия

эквивалентна

минимизации `log-loss`.

Это и есть главное теоретическое обоснование вашей реализации.

---

## 6. Почему это действительно эквивалентно

Пусть:

`f(theta) = log L(theta)`

Тогда:

- если `theta` максимизирует `f(theta)`,
- то тот же `theta` минимизирует `-f(theta)`.

А деление на `m`:

- просто умножение на положительную константу `1/m`;
- оно не меняет точку минимума.

Значит:

`argmax log L(theta) = argmin [ - (1/m) log L(theta) ]`

То есть:

`argmax L(theta) = argmin log-loss`

Именно поэтому можно уверенно говорить:

"Я использую метод максимального правдоподобия, потому что минимизирую `log-loss`."

---

## 7. Подготовка к выводу градиента

Для удобства обозначим:

`z_i = b + w^T x_i`

`p_i = sigma(z_i)`

Функция потерь:

`J(w, b) = - (1 / m) * Sum(i = 1..m) [ y_i log p_i + (1 - y_i) log(1 - p_i) ]`

Нужно найти производные по параметрам.

Сначала полезно помнить:

`sigma'(z) = sigma(z)(1 - sigma(z))`

То есть:

`d p_i / d z_i = p_i (1 - p_i)`

---

## 8. Производная по линейному отклику `z_i`

Возьмём один член суммы:

`l_i = - [ y_i log p_i + (1 - y_i) log(1 - p_i) ]`

Найдём производную по `z_i`.

Сначала по `p_i`:

`d l_i / d p_i = - [ y_i / p_i - (1 - y_i) / (1 - p_i) ]`

Теперь используем цепное правило:

`d l_i / d z_i = (d l_i / d p_i) * (d p_i / d z_i)`

Подставляем:

`d l_i / d z_i = - [ y_i / p_i - (1 - y_i) / (1 - p_i) ] * p_i (1 - p_i)`

Раскрываем:

`d l_i / d z_i = - [ y_i (1 - p_i) - (1 - y_i) p_i ]`

Упростим:

`d l_i / d z_i = - [ y_i - y_i p_i - p_i + y_i p_i ]`

`d l_i / d z_i = - [ y_i - p_i ]`

`d l_i / d z_i = p_i - y_i`

Это очень важный результат:

производная по линейному отклику равна

`p_i - y_i`

То есть "предсказание минус истина".

---

## 9. Производная по весам

Поскольку:

`z_i = b + w^T x_i`

то для каждого веса `w_j`:

`d z_i / d w_j = x_{ij}`

Теперь снова используем цепное правило:

`d l_i / d w_j = (d l_i / d z_i) * (d z_i / d w_j)`

Подставляем:

`d l_i / d w_j = (p_i - y_i) x_{ij}`

Теперь для всей функции потерь:

`dJ / dw_j = (1 / m) * Sum(i = 1..m) [ (p_i - y_i) x_{ij} ]`

Это производная по одному весу.

---

## 10. Производная по свободному члену `b`

Для свободного члена:

`d z_i / d b = 1`

Тогда:

`d l_i / d b = (p_i - y_i) * 1 = p_i - y_i`

И для всей выборки:

`dJ / db = (1 / m) * Sum(i = 1..m) [ p_i - y_i ]`

Это та же логика, просто без множителя `x_{ij}`.

---

## 11. Векторная форма градиента

Если записать всё сразу в матричном виде, получаем:

`grad = (1 / m) * X^T (p - y)`

Где:

- `X` — матрица признаков;
- `p` — вектор вероятностей;
- `y` — вектор истинных меток;
- `X^T` — транспонированная матрица признаков.

Если в `X` уже добавлен столбец единиц для bias, то:

- в одном векторе градиента сразу содержатся производные и по `b`, и по всем `w_j`.

Это именно та запись, которая реализована у вас в коде.

---

## 12. Шаг градиентного спуска

После того как градиент найден, параметры обновляются:

`theta := theta - alpha * grad`

Где:

- `theta` — вектор всех параметров;
- `alpha` — скорость обучения.

Для вашей модели:

`w := w - alpha * (1 / m) * X^T (p - y)`

Смысл:

- если градиент показывает направление роста функции,
- то минус градиент ведёт к уменьшению функции.

Поэтому после многих шагов модель движется к минимуму `log-loss`.

---

## 13. Почему остановка по `tolerance` разумна

На итерации `t` есть значение:

`J_t`

На прошлой итерации было:

`J_{t-1}`

Если:

`|J_t - J_{t-1}| < tolerance`

то изменение настолько маленькое, что можно считать:

- существенного улучшения уже нет;
- алгоритм практически сошёлся.

Это рабочий критерий остановки.

---

## 14. Формула стандартизации

Для каждого признака `x_j`:

`x'_j = (x_j - mu_j) / sigma_j`

Где:

- `mu_j` — среднее по признаку;
- `sigma_j` — стандартное отклонение по признаку.

После этого:

- среднее становится около `0`;
- масштаб становится сопоставимым.

Это улучшает поведение градиентного спуска.

---

## 15. Формула accuracy

Пусть:

- `y_i` — истинный класс;
- `y_hat_i` — предсказанный класс.

Тогда:

`accuracy = (1 / m) * Sum(i = 1..m) I(y_hat_i = y_i)`

Где:

- `I(...)` это индикатор:
- он равен `1`, если условие истинно;
- и `0`, если ложно.

То есть accuracy это просто доля правильных ответов.

---

## 16. Формула корреляции Пирсона

Для двух переменных `x` и `y`:

`r_xy = Sum(i = 1..m) [ (x_i - x_mean)(y_i - y_mean) ] / sqrt( Sum(i = 1..m)(x_i - x_mean)^2 * Sum(i = 1..m)(y_i - y_mean)^2 )`

Смысл:

- числитель показывает совместное отклонение;
- знаменатель нормирует результат.

После нормировки:

- `r` всегда лежит в диапазоне `[-1, 1]`.

В отборе признаков используется:

`|r_xy|`

потому что важна сила связи.

---

## 17. Формула наивного отбора признаков

Для каждого признака `x_j` считаем:

`score_j = |corr(x_j, y)|`

Затем:

1. сортируем все `score_j` по убыванию;
2. берём 6 лучших признаков.

Это и есть наивный отбор.

Он учитывает:

- только связь признака с целевой переменной.

Он не учитывает:

- взаимную корреляцию между признаками.

---

## 18. Формула критерия CFS

Для подмножества из `k` признаков:

`Merit = (k * r_cf) / sqrt(k + k(k - 1) r_ff)`

Где:

- `r_cf` — средняя корреляция признаков с целевой переменной;
- `r_ff` — средняя корреляция признаков друг с другом.

### Интерпретация

Числитель:

- поощряет информативность.

Знаменатель:

- штрафует избыточность.

Поэтому CFS выбирает не просто "сильные" признаки, а "хороший набор" признаков.

---

## 19. Как объяснить всё это устно очень кратко

Если преподаватель просит краткий, но строгий ответ, можно сказать так:

"Логистическая регрессия моделирует вероятность класса `1` как `sigma(b + w^T x)`. Для бинарного ответа вероятность одного наблюдения записывается как `p^y (1-p)^{1-y}`. При предположении независимости всей выборки получаем функцию правдоподобия как произведение этих вероятностей. После логарифмирования произведение превращается в сумму, и максимизация правдоподобия сводится к минимизации отрицательного среднего логарифма правдоподобия, то есть `log-loss`. Градиент этой функции в матричной форме равен `(1/m) X^T (p-y)`, поэтому параметры обновляются шагом `w := w - alpha * grad`. Для отбора признаков используется либо наивный критерий по `|corr(feature, target)|`, либо CFS, где максимизируется средняя связь с классом и одновременно штрафуется взаимная корреляция признаков."

---

## 20. Самое главное, что нужно запомнить

Если времени мало, запомни именно эту логическую цепочку:

1. `z = b + w^T x`
2. `p = sigma(z)`
3. `P(y|x) = p^y (1-p)^{1-y}`
4. `L = Product P(y_i|x_i)`
5. `log L = Sum [ y_i log p_i + (1-y_i) log(1-p_i) ]`
6. `log-loss = - (1/m) log L`
7. `grad = (1/m) X^T (p-y)`
8. `w := w - alpha * grad`

Это вся математическая основа вашей лабораторной в одной цепочке.
