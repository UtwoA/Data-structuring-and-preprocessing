## Строки 67-107: класс логистической регрессии

`67: class LogisticRegressionGD:`

- Объявляет класс модели.
- `GD` означает `Gradient Descent`.

### Инициализация

`68: def __init__(self, learning_rate=0.1, max_iterations=20000, tolerance=1e-8):`

- Конструктор класса.
- Принимает гиперпараметры обучения.

`69: self.learning_rate = learning_rate`

- Сохраняет скорость обучения внутрь объекта.

`70: self.max_iterations = max_iterations`

- Сохраняет максимальное число итераций.

`71: self.tolerance = tolerance`

- Сохраняет порог остановки.

`72: self.weights = None`

- Пока веса модели не обучены, они пустые.

### Обучение

`74: def fit(self, features, target):`

- Метод обучения модели.
- Именно здесь реализованы максимальное правдоподобие и градиентный спуск.

`75: sample_count, feature_count = features.shape`

- Получает число объектов и число признаков.

`76: design_matrix = np.hstack((np.ones((sample_count, 1)), features))`

- Добавляет слева столбец единиц.
- Это нужно для свободного коэффициента модели.

`78: self.weights = np.zeros(feature_count + 1, dtype=np.float64)`

- Инициализирует все веса нулями.
- Весов на один больше, потому что есть bias.

`79: previous_loss = np.inf`

- Предыдущее значение потерь задаётся как бесконечность.
- Это нужно для первой проверки сходимости.

`81: for _ in range(self.max_iterations):`

- Запускает цикл градиентного спуска.

`82: linear_response = design_matrix @ self.weights`

- Считает линейную комбинацию признаков `Xw`.

`83: probabilities = sigmoid(linear_response)`

- Преобразует линейный отклик в вероятности класса 1.

`85: epsilon = 1e-12`

- Малое число для численной устойчивости.

`86-89: loss = -np.mean(...)`

- Считает log-loss.
- Это отрицательный логарифм правдоподобия.
- Именно тут в коде реализован метод максимального правдоподобия в вычислительной форме.

`91: gradient = (design_matrix.T @ (probabilities - target)) / sample_count`

- Считает градиент функции потерь по всем весам.

`92: self.weights -= self.learning_rate * gradient`

- Обновляет веса в направлении уменьшения функции потерь.
- Это и есть шаг градиентного спуска.

`94: if abs(previous_loss - loss) < self.tolerance:`

- Проверяет, насколько изменился loss по сравнению с прошлой итерацией.

`95: break`

- Если изменение слишком мало, обучение останавливается.

`97: previous_loss = loss`

- Запоминает текущий loss как предыдущий для следующей итерации.

`99: return self`

- Возвращает сам объект модели.
- Это позволяет вызывать методы цепочкой и вообще удобно по стилю.

### Предсказание вероятности

`101: def predict_proba(self, features):`

- Метод для предсказания вероятностей.

`102: design_matrix = np.hstack((np.ones((features.shape[0], 1)), features))`

- Снова добавляет столбец единиц для свободного коэффициента.

`103: return sigmoid(design_matrix @ self.weights)`

- Считает вероятности класса 1.

### Предсказание класса

`105: def predict(self, features, threshold=0.5):`

- Метод для получения уже не вероятностей, а классов `0` или `1`.

`106: probabilities = self.predict_proba(features)`

- Сначала получает вероятности.

`107: return (probabilities >= threshold).astype(np.int64)`

- Применяет порог `0.5`.
- Всё, что не меньше порога, становится классом `1`.

## Строки 147-162: критерий CFS

`147: def cfs_merit(feature_indices, feature_target_corr, feature_feature_corr):`

- Объявляет функцию оценки подмножества признаков по CFS.

`148: subset_size = len(feature_indices)`

- Считает число признаков в выбранном подмножестве.

`149: mean_feature_class_corr = float(np.mean(feature_target_corr[list(feature_indices)]))`

- Считает среднюю корреляцию признаков с целевой переменной.

`151: pair_corr = []`

- Создаёт список парных корреляций внутри подмножества.

`152: for first_index, second_index in combinations(feature_indices, 2):`

- Перебирает все пары признаков внутри выбранного подмножества.

`153: pair_corr.append(feature_feature_corr[first_index, second_index])`

- Сохраняет корреляцию каждой пары.

`155: mean_feature_feature_corr = float(np.mean(pair_corr)) if pair_corr else 0.0`

- Считает среднюю взаимную корреляцию признаков.

`156-158: denominator = np.sqrt(...)`

- Вычисляет знаменатель формулы CFS.

`159: if denominator == 0:`

- Защита от деления на ноль.

`160: return 0.0`

`162: return (subset_size * mean_feature_class_corr) / denominator`

- Возвращает итоговое значение `Merit`.
- Чем оно больше, тем лучше подмножество.

## Почему график сходимости удалён

Ранее в проекте был дополнительный график `baseline_loss.png`.

Почему его убрали:

- он не требуется ТЗ;
- он не влияет на итоговую метрику;
- он не нужен для сравнения моделей;
- для простой учебной защиты он только добавляет лишние вопросы.

То есть:

- тепловая карта нужна, потому что она прямо требуется в задании;
- график сходимости не нужен, потому что это уже вспомогательная диагностика, а не обязательный результат.
